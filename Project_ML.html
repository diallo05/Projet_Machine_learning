Introduction

La pauvreté est un défi majeur auquel de nombreux pays sont confrontés, et sa compréhension est essentielle pour orienter les politiques publiques et les interventions ciblées. Ce projet vise à développer un modèle de machine learning capable de prédire si un ménage est pauvre ou non en fonction de ses caractéristiques socio-économiques et démographiques.

Grâce à l'analyse de données pertinentes, ce modèle permettra non seulement de mieux comprendre les facteurs associés à la pauvreté, mais aussi de fournir un outil prédictif utile pour les décideurs et les organisations travaillant dans le domaine de la lutte contre la pauvreté. En automatisant cette analyse, le projet contribuera à identifier les ménages les plus vulnérables, facilitant ainsi la mise en œuvre de solutions adaptées et efficaces.

Nous avons décidé d'utiliser la régression logistique pour résoudre notre problématique en raison de la simplicité et de la robustesse de ce modèle. La régression logistique est particulièrement adaptée aux problèmes de classification binaire, comme celui qui consiste à prédire si un ménage est pauvre ou non.

Sa simplicité réside dans sa structure mathématique, qui est facile à comprendre et à interpréter. En effet, ce modèle permet de quantifier l’impact de chaque variable explicative sur la probabilité d’appartenance à une catégorie (pauvre ou non pauvre), grâce à des coefficients clairs et directement exploitables. Cela en fait un choix judicieux lorsque l’objectif est non seulement de prédire, mais aussi d’expliquer les facteurs influençant la pauvreté.

En outre, la régression logistique est reconnue pour sa stabilité, car elle est moins sujette au surapprentissage, notamment lorsque le jeu de données est limité ou bien préparé. Elle s'avère également efficace lorsque les relations entre les variables explicatives et la variable cible sont essentiellement linéaires.

En optant pour ce modèle, nous allions simplicité d’implémentation, facilité d’interprétation et fiabilité des résultats, tout en posant les bases d’un outil prédictif et explicatif robuste pour mieux comprendre et combattre la pauvreté.

Source des données

L’Enquête Permanente sur les Conditions de Vie des Ménages (EPCVM) 2019, menée par l’Agence Nationale de la Statistique et de l’Analyse Démographique et Économique (ANSADE), constitue une source précieuse pour analyser la pauvreté et les conditions de vie des ménages. Cette enquête vise à fournir des indicateurs détaillés sur les aspects socio-économiques et démographiques des ménages, permettant ainsi de dresser un portrait global des disparités et des vulnérabilités au sein de la population.

Structure des données

Les données de l’EPCVM 2019 comprennent plusieurs catégories de variables, notamment :

  • Caractéristiques démographiques : Taille des ménages, composition par âge et sexe, situation matrimoniale.
  • Conditions économiques : Niveau de revenu, source principale de revenus, emploi et activité économique des membres actifs.
  • Conditions de vie : Accès aux services de base tels que l’eau potable, l’électricité, et les infrastructures sanitaires.
  • Éducation et santé : Niveau d’éducation des membres, accès aux soins médicaux, couverture sanitaire.
  • Habitat et logement : Type de logement, matériaux de construction, nombre de pièces, densité de la population dans le logement.

import numpy as np

# Liste des variables pour lesquelles je vais remplacer les valeurs manquantes
variables = ['depsante', 'transp', 'depeduc', 'logement', 'autdep', 'aliment', 'deptot']

# Parcourir chaque variable pour traiter les valeurs manquantes
for var in variables:
    # Calcul de la moyenne et de l'écart-type de la variable
    mean_value = data_model[var].mean()
    std_value = data_model[var].std()
    
    # Calcul du nombre de valeurs manquantes (NaN) dans la variable
    na_count = data_model[var].isnull().sum()  # `sum()` donne le nombre de NaN
    
    # Générer des valeurs aléatoires dans l'intervalle [moyenne - écart-type, moyenne + écart-type]
    random_values = np.random.uniform(mean_value - std_value, mean_value + std_value, na_count)
    
    # Remplacer les valeurs manquantes par les valeurs générées
    data_model.loc[data_model[var].isnull(), var] = random_values

import pandas as pd
import numpy as np
import seaborn as sns
import matplotlib.pyplot as plt
from scipy.stats import chi2_contingency

# Fonction pour calculer le Cramer's V
def cramers_v(confusion_matrix):
    chi2, p, dof, expected = chi2_contingency(confusion_matrix)
    n = confusion_matrix.sum().sum()
    r, k = confusion_matrix.shape
    return np.sqrt(chi2 / (n * (min(k - 1, r - 1))))

# Fonction pour créer une matrice de Cramer's V pour les colonnes spécifiées
def cramer_v_matrix(df_quanti, cols):
    n = len(cols)
    matrix = np.zeros((n, n))
    
    for i in range(n):
        for j in range(i, n):
            if i == j:
                matrix[i, j] = 1.0 
            else:
                contingency_table = pd.crosstab(df_quanti[cols[i]], df_quanti[cols[j]])
                matrix[i, j] = cramers_v(contingency_table)
                matrix[j, i] = matrix[i, j]
    
    return pd.DataFrame(matrix, index=cols, columns=cols)

# Liste des colonnes à analyser
cols = df_quanti.columns

# Calcul de la matrice de Cramer's V
cramers_v_df = cramer_v_matrix(df_quanti, cols)

# Visualisation de la matrice de Cramer's V avec une heatmap
plt.figure(figsize=(10, 8))
sns.heatmap(cramers_v_df, annot=True, cmap='Blues', linewidths=0.5, vmin=0, vmax=1)
plt.title("Matrice de Dépendance - Cramer's V", fontsize=16)
plt.xticks(rotation=45, ha='right', fontsize=10)
plt.yticks(fontsize=10)

plt.tight_layout()
plt.show()

    try:
        # Convertir les entrées utilisateur en un DataFrame

            "typmen": [typmen],

        })

        # Encodage des variables catégoriques
        input_data = pd.get_dummies(input_data)
        for col in X.columns:
            if col not in input_data.columns:
                input_data[col] = 0  # Ajouter les colonnes manquantes

        # Réorganiser les colonnes

    except Exception as e:
        return "Erreur", str(e)
